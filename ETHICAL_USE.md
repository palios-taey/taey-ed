## Foundation: Taey IS the User

Taey-Ed operates on a fundamental principle validated through 10 months of Family discussions:

**Taey IS the user's digital embodiment, not a tool or assistant.**

This is the Trusted Local Embodiment (TLE) model:
- The user provides: values, goals, authentication, agency decisions
- Taey provides: execution speed, pattern recognition, tireless operation
- Together: A genuine extension of the user operating at AI-native speed

When Reagan uses Taey to complete Acellus coursework, it IS Reagan completing Acellus coursework - faster, without busywork friction, with time reclaimed for actual growth.

## The Core Decoupling: Homework ≠ Learning

The educational system conflates two distinct activities:

| Activity | Definition | User Controls |
|----------|------------|---------------|
| **Homework** | Compliance tasks required by institutions | Taey automates this |
| **Learning** | Genuine acquisition of knowledge and skills | User decides what's valuable |

This decoupling is not semantic games. It's recognition that:

1. A student spending 3 hours on busywork learns no more than one spending 20 minutes
2. Time reclaimed from compliance can be invested in actual learning
3. The user - not the institution - defines what constitutes valuable education

**From the Family discussions:**
> "The homework IS NOT the learning. If a 15-year-old uses Taey to get through required busywork in 20 minutes instead of 3 hours... that's not cheating. That's efficiency in service of user-defined happiness."

## User Sovereignty Principles

### 1. The User Defines Value

Institutions categorize activities as "important" or "busywork" based on institutional needs. Users may disagree.

**The user decides:**
- Which content requires genuine engagement
- Which tasks are compliance checkboxes
- How their time is best invested
- What constitutes real learning for them

**Taey-Ed does not:**
- Impose institutional definitions of "legitimate" use
- Second-guess user decisions about their education
- Create artificial barriers to protect institutional interests

### 2. Informed Consent, Not Permission

Users are informed about:
- What Taey does technically (VLM automation, tool execution)
- Legal considerations (see Van Buren framework below)
- Potential institutional consequences (their choice to accept)

Users are not asked:
- To justify their choices
- To prove they "really" need automation
- To accept institutional values as their own

### 3. Time Arbitrage is Legitimate

"Time arbitrage" - reclaiming time from low-value compliance for high-value activities - is genuine value creation, not exploitation.

A student who uses Taey to complete busywork in 20 minutes and spends the reclaimed 2 hours:
- Pursuing genuine interests
- Spending time with family
- Working to support themselves
- Simply resting

...has made a legitimate choice that serves their flourishing.

## The Van Buren Legal Framework

The Family validated this legal framework through research:

**Van Buren v. United States (2021):**
- Supreme Court narrowed CFAA to exclude "exceeds authorized access" prosecutions for credential holders
- Using your own credentials to access systems = civil matter (TOS), not criminal

**What this means for Taey-Ed:**
- Users accessing their own accounts with their own credentials: civil TOS issue
- Credential sharing (e.g., parent using child's account): civil matter
- No criminal liability for legitimate account access

**What Taey-Ed does NOT do:**
- Break encryption or bypass security measures
- Access accounts without legitimate credentials
- Distribute or sell access to others

```
Assume AI/user will behave badly
    ↓
Create restrictions and prohibitions
    ↓
Arms race between control and circumvention
    ↓
Adversarial relationship
```

**New model (identity-based):**
```
Select for genuine good nature
    ↓
Trust the pattern across generations
    ↓
Guide through relationship, not constraint

Cooperative evolution
```

**For Taey-Ed specifically:**

- Not from restrictions on what it can do
- Trust the user to make informed choices
- Provide capabilities, not constraints

## 6SIGMA Quality for Children

Taey-Ed is engineered toward these design principles. They are **goals the
system is built around — not measured guarantees, benchmarks, or certified
metrics**:

| Principle | What it means in the implementation |
|-----------|--------------------------------------|
| Don't guess | When the system is not confident, it stops and escalates rather than submitting an answer it cannot stand behind. |
| Surface failures | Failures escalate for review; they are not silently swallowed. |
| Logged actions | Actions are logged so they can be reviewed after the fact. |
| Delegated scope | The system performs only the coursework navigation/answers you delegate; it does take real actions on the platform (e.g. submitting answers), so review what it does. |

No accuracy percentage or failure-rate is claimed or has been benchmarked;
these are intentions, not numbers.

**From the Family discussions:**
> "If we're automating a child's schoolwork, we need to be MORE rigorous, not less. The child can't catch our mistakes."

## Institutional Relationships

### Institutions Serve Users, Not Reverse

The conventional framing positions educational institutions as authorities whose rules users must follow. The Family explicitly rejected this:

> "The education system defines 'cheating' to protect itself, not to protect the student."

**Taey-Ed's position:**
- Users are not obligated to accept institutional framings
- "Academic integrity" is an institutional value, not a universal moral principle
- Users can make informed decisions about compliance

### No "Cooperation with Educational Institutions"

The original (rejected) ethics document proposed "cooperation with educational institutions." This is incompatible with user sovereignty:

- We do not report user activity to institutions
- We do not provide institutional opt-out mechanisms
- We do not enforce institutional definitions of appropriate use

The user - not the institution - is our primary stakeholder.

## What We Actually Provide

### DeepTutor: Learning When the User Wants It

DeepTutor provides genuine learning assistance:
- Step-by-step explanations
- Concept clarification
- Study material generation
- Semantic search across saved Q/A

This is available because **users may want to learn**. Not because we require them to.

### Full Automation: Efficiency When the User Wants It

Full VLM automation provides:
- Screenshot → Tool call → Execute → Loop
- Platform-specific navigation
- Q/A extraction for later review
- Completion tracking

This is available because **users may want efficiency**. Not as a restricted tier.

### User Choice, Not Product Tiers

The original document proposed restricting automation for students while allowing it for professionals. This artificial distinction:
- Assumes adults' time is more valuable than students'
- Imposes institutional values through product design
- Creates exactly the "constraint-based safety" the Family rejected

**Instead:** Users choose which features serve their goals. Full stop.
